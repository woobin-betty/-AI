## 1. 프로젝트 개요

사용자의 과제·업무 목록과 일정 데이터를 분석해 우선순위를 자동 정리하고, 팀 프로젝트의 일정 조율과 진행 관리까지 지원하는 스마트 일정 관리 서비스

---

## 2. 문제 정의 및 솔루션

### 2.1 핵심 문제

- 해야 할 과제/업무의 **마감일과 우선순위를 한눈에 관리하기 어려움**
    - 마감일을 놓치거나, 중요도가 낮은 일부터 시작해 비효율 발생
- 팀 프로젝트의 경우 **팀원 간 일정 조율과 진행 상황 공유가 번거로움**
    - 각자 바쁜 일정으로 회의 시간을 잡기 어렵고, 작업 분담이 명확하지 않아 갈등 발생
- 작업 진행 중 **체계적인 체크리스트나 단계 가이드가 없어 시간 관리가 어려움**

### 2.2 솔루션 가설

**"마감 일정 자동 정리, 팀 일정 조율 기능, 단계별 체크 가이드를 제공하는 일정 관리 플랫폼은 과제/업무 효율을 높이고 스트레스를 줄일 것이다."**

---

## 3. 핵심 기능

### (1) AI 기반 과제 일정 자동 생성

과제 이름과 마감일을 입력하면 AI가 과제를 세부 단계로 분할하고, 하루 단위 계획표를 자동으로 생성

### (2) **진행 현황 시각화 및 리마인드 알림**

진행률을 그래프 형태로 확인 가능하며, 마감 전 주요 단계별 알림과 일정 리마인드 기능 제공

### (3) **사용자 맞춤 학습 패턴 분석**

사용자의 작업 속도, 집중 시간대, 과제 유형 데이터를 분석하여 개인별 최적 일정과 작업 루틴 추천

---

## 4.사용자 시나리오

**과제, 업무, 학습 일정을 함께 관리해야 하는 사용자**

1. 사용자가 해야 할 일과 마감 기한을 앱에 입력
    
    (예: *‘회사 보고서 제출’, ‘온라인 강의 수강’, ‘자격증 과제 업로드’*)
    
2. 시스템이 **중요도 / 마감일 / 예상 소요시간**을 분석하여
    
    **자동으로 일정 우선순위를 정렬**
    
    → 먼저 해야 할 작업과 여유 있는 작업을 한눈에 확인 가능
    
3. **사용자의 작업 일정을 시각적으로 확인할 수 있도록
서비스 내부에 캘린더 UI 제공**
    - 생성된 과제 일정과 단계별 진행 계획을 **캘린더에서 한눈에 확인**
    - **일 / 주 / 월 단위 보기 전환** 지원
    - 일정 카드 클릭 시 **세부 단계 / 체크리스트 / 진행률** 확인 가능
    - 필요할 경우 **드래그 앤 드롭으로 일정 시간대 재조정** 가능
    
    **예:**
    
    “과제 계획 생성 → 자동으로 3일 분량으로 일정 배치 →
    
    사용자가 원하는 날짜로 이동 / 조정 가능”
    
4. 진행 중인 작업은 **체크리스트 형태로 단계별 가이드 제공**
5. **진행률 표시 + 다음에 해야 할 단계 안내**
6. 마감일이 다가오면
    
    **알림 + 제출 전 점검 목록 자동 제공**
    
    (예: *‘파일 형식 확인’, ‘첨부 누락 여부 확인’ 등)
    

→ 사용자는 체계적으로 일정과 과제를 관리할 수 있어

**마감 스트레스가 감소하고, 제출/완료율이 향상됨.**

## 5. 기대 효과

### 사용자 측면

- **시간 관리 능력 향상:** AI가 자동으로 계획을 세워주므로 사용자는 효율적인 학습 루틴을 형성할 수 있습니다.
- **마감 스트레스 감소:** 과제 진행 상황을 시각적으로 확인하고, 하루 단위 목표를 제시받아 심리적 부담이 줄어듭니다.
- **생산성 및 성취감 향상:** 계획에 따라 꾸준히 과제를 완성함으로써 성취감과 자기효능감이 높아집니다.

### 사회적 측면

- **학습 문화 개선:** 벼락치기 중심의 비효율적인 학습 문화를 개선하고, 체계적인 자기주도 학습 문화를 조성합니다.
- **정신건강 증진:** 과제 스트레스와 불안감 감소로 대학생들의 전반적인 정신적 안정에 기여합니다.
- **디지털 교육 혁신 촉진:** AI를 활용한 학습 지원 서비스 확산으로 교육 기술의 발전과 사회적 수용도를 높입니다.

### 비즈니스 측면

- **에듀테크 시장 경쟁력 확보:** 대학생 특화 AI 일정 관리라는 틈새 시장을 선점할 수 있습니다.
- **프리미엄 서비스 확장 가능성:** 맞춤형 학습 코칭, 팀 프로젝트 일정 관리 등 유료 부가 서비스로 수익 창출이 가능합니다.
- **데이터 기반 서비스 고도화:** 사용자 패턴 데이터를 통해 학습 분석 및 예측 모델을 발전시켜 장기적 비즈니스 가치 확보가 가능합니다.

    모바일 우선 웹 서비스(SPA) 요구사항과 Spotify API 연동, 세션 관리를 고려한 기술 스택 
    
    ## 1. 핵심 프레임워크 (Full-stack)
    
    - **Next.js** (React 18+ 기반)
    
    ## 2. 프론트엔드 (Frontend)
    
    - **Tailwind CSS**
    - **Lucide React**
    - **TanStack Query (React Query)**
    - **Zustand** (상태 관리)
    
    ## 3. 백엔드 (Backend)
    
    - **Next.js API Routes**
    - **Auth.js (구 Next-Auth)**
    - **Spotify Web API**
    
    ## 4. 데이터베이스
    
    - **Vercel KV** (Serverless Redis)
    
    ## 5. 배포 (Deployment)
    
    - **Vercel**

    ### 1. AI 모델 (Generative AI Core)
    
    - **주요 기술:** 대규모 언어 모델 (LLM) API 연동
        - **선택:** **OpenAI GPT-4 또는 GPT-3.5 Turbo** (레시피 생성, 재료 분석, 조리 단계 상세화에 최적)
        - **대안:** Google Gemini API, Anthropic Claude API (성능 및 비용 비교 후 선택 가능)
    - **역할:**
        - 사용자가 입력한 재료를 기반으로 독창적이고 실용적인 레시피를 생성
        - 생성된 레시피의 요리명, 조리 시간, 난이도, 필요한 재료 목록, 단계별 조리법을 구조화된 형식으로 제공
        - (선택적) 레시피 생성 불가능 시 사용자에게 친화적인 대체 제안 또는 추가 재료 추천
    
    ---
    
    ### 2. 프론트엔드 (User Interface & Experience)
    
    - **주요 기술:** SPA(Single Page Application) 프레임워크
        - **선택:** **React** (활발한 커뮤니티, 풍부한 라이브러리, 컴포넌트 기반 개발 용이)
        - **대안:** Vue.js (학습 곡선이 낮고 빠른 개발), Svelte (빌드 시 최적화, 높은 성능)
    - **UI 라이브러리:**
        - **선택:** **Tailwind CSS** (유틸리티-퍼스트 CSS 프레임워크로 빠른 스타일링 및 커스터마이징 용이) 또는 **Chakra UI / Ant Design** (미리 빌드된 컴포넌트로 빠른 UI 구성)
    - **역할:**
        - 직관적인 재료 입력 필드 및 '레시피 생성' 버튼 제공
        - 생성된 레시피를 가독성 좋고 구조화된 형태로 출력 (요리명, 재료, 단계별 조리법 등)
        - 로딩 스피너, 에러 메시지 등 사용자 피드백을 실시간으로 제공
        - 반응형 디자인 구현으로 다양한 디바이스(모바일, 태블릿, 데스크톱) 지원
    
    ---
    
    ### 3. 백엔드 (API & LLM Integration)
    
    - **주요 기술:** 경량 웹 프레임워크 또는 서버리스 함수
        - **선택 (Python):** **FastAPI** (높은 성능, 비동기 지원, 자동 문서화) 또는 **Flask** (가볍고 유연)
        - **선택 (Node.js):** **Express.js** (빠른 API 개발, JavaScript 생태계 활용)
    - **역할:**
        - 프론트엔드로부터 재료 입력 요청 수신
        - LLM API와 연동하여 레시피 생성 요청 및 응답 처리
        - LLM 응답을 파싱하여 프론트엔드에서 사용하기 쉬운 JSON 형태로 가공 및 전송
        - (MVP 이후) 사용자 인증, 레시피 저장/조회 등 데이터베이스 연동 로직 처리
    
    ---
    
    ### 4. 데이터베이스 (Optional for MVP, Essential for Persistence)
    
    - **필요성:** MVP 단계에서는 레시피 저장 기능이 필수는 아니지만, '즐겨찾기'나 '나의 레시피' 같은 기능을 추가하려면 필요.
    - **주요 기술:** NoSQL 데이터베이스 (유연한 스키마, 빠른 개발)
        - **선택:** **Firebase Firestore** (서버리스 백엔드와의 연동 용이, 실시간 동기화 지원)
        - **대안:** MongoDB (오픈 소스, 확장성), DynamoDB (AWS 환경에서 사용 시)
    - **역할:**
        - 사용자가 즐겨찾기한 레시피 저장 및 조회
        - (향후) 사용자 계정 정보, 선호도 데이터 저장
    
    ---
    
    ### 5. 배포 (Deployment & Hosting)
    
    - **주요 기술:** 서버리스 플랫폼 또는 클라우드 서비스
        - **선택 (프론트엔드):** **Vercel** 또는 **Netlify** (Git 연동을 통한 자동 배포, CDN 지원, 무료 티어 제공)
        - **선택 (백엔드):** **Google Cloud Functions / AWS Lambda / Azure Functions** (서버리스 함수로 LLM API 호출 및 응답 처리, 비용 효율적)
        - **대안 (통합):** Render.com, Heroku (빠른 프로토타이핑 및 배포)
    - **역할:**
        - 개발된 애플리케이션(프론트엔드 및 백엔드)을 안정적으로 운영
        - 자동 스케일링을 통해 사용자 트래픽 변화에 유연하게 대응
        - CI/CD (Continuous Integration/Continuous Deployment) 파이프라인 구축 (선택)

### 1. AI 모델 (Generative AI Core)

- **주요 기술:** 대규모 언어 모델 (LLM) API 연동
- **선택:** **Anthropic Claude API** (과제 설명 분석, 수행 단계 가이드 생성에 최적)
- **대안:** OpenAI GPT-5 / GPT-4 API, Google Gemini API (성능 및 비용 비교 후 선택 가능)
- **역할:**
    - 사용자가 입력한 과제명, 설명, 마감일을 분석하여 **과제의 유형 및 난이도 자동 파악**
    - 해당 과제를 수행하기 위한 **단계별 실행 계획, 시간 배분, 제출 체크리스트 생성**
    - 과제 난이도 또는 사용자의 여유 시간에 따라 **일정 계획 자동 조정**
    - (선택적) 과제를 수행하는 데 필요한 참고자료 또는 검색 키워드 제안

---

### 2. 프론트엔드 (User Interface & Experience)

- **주요 기술:** 웹 기반 UI 빌더
    - **선택:** **Clude App Builder** (코딩 없이 화면 구성 가능)
- **UI 라이브러리:** 자체 제공 기본 컴포넌트 활용
- **역할:**
    - 과제 입력 Form UI 제공 (과제명, 설명, 마감일, 메모 입력)
    - 생성된 AI 가이드를 **카드 / 리스트 형태**로 출력
    - 과제 클릭 시 단계별 진행 가이드 상세 페이지 또는 모달 표시
    - 각 단계를 체크할 수 있는 **진행 현황 표시 기능 제공**
    - 모바일 및 데스크톱 환경에서의 **반응형 인터페이스 자동 지원**

---

### 3. 백엔드 (API & LLM Integration)

- **주요 기술:** 서버리스 API 실행 방식
    - **선택:** **Clude 내장 API 액션 기능**
- **역할:**
    - 프론트엔드에서 전달된 과제 입력 데이터를 수신
    - Claude API에 요청을 전달하여 AI 수행계획 생성
    - LLM 모델의 응답을 파싱하여 프론트에서 사용 가능한 JSON 형식으로 변환
    - (MVP 이후) 사용자별 과제 기록 조회 / 수정 / 삭제 로직 추가 가능

---

### 4. 데이터베이스 (Optional for MVP, Essential for Persistence)

- **필요성:**
    
    MVP 단계에서는 단순 실행 가능하지만, **과제 저장 / 진행률 저장** 기능을 위해 데이터베이스가 필요함.
    
- **주요 기술:** 서버리스 NoSQL 데이터 저장
    - **선택:** **Clude Database (내장 테이블 저장 기능)**
- **역할:**
    - 사용자별 과제 목록 저장 및 조회
    - 단계별 완료 여부(체크박스 기반) 저장
    - (향후) 사용자 맞춤 학습 패턴 분석 기능에 활용 가능

---

### 5. 배포 (Deployment & Hosting)

- **주요 기술:** 서버리스 플랫폼
    - **선택:** **Clude 자동 배포 시스템** (별도 서버 설정 불필요)
- **역할:**
    - 프론트엔드 / 백엔드 / DB가 통합된 환경에서 일괄 운영
    - Git 없이도 실시간 버전 업데이트 및 롤백 가능
    - 사용자 증가 시 자동 확장 스케일링 지원
